### Вариант 7
Проект представляет собой консольное приложение-симулятор банкомата

### Использованный стек
Cats, Cats Effect, ZIO не используются, работа построена на стандартной библиотеке Scala

### Основные возможности

- **Авторизация и сессии:** Переключение между аккаунтами пользователей по имени.
- **Снятие наличных:** Алгоритм выдачи денег минимальным количеством купюр с проверкой баланса и дневных лимитов.
- **Автоматическое округление:** Логика округления запрашиваемой суммы под доступные номиналы купюр.
- **Пополнение счета:** Безопасное обновление баланса в иммутабельном состоянии.
- **Денежные переводы:** Переводы между пользователями с автоматическим расчетом комиссии.
- **Симуляция смены дней:** Сброс дневных лимитов на снятие наличных.
- **Логирование (Чек операции):** Накопление детальных логов каждого шага логики и их вывод в виде чека.

## Инфраструктура
### Реализованные монады:

1. **Интерфейс вместо использования сторонних библиотек:**

```
trait Monad[M[_]]:
  def pure[A](a: A): M[A]
  def flatMap[A, B](ma: M[A])(f: A => M[B]): M[B]
  def map[A, B](ma: M[A])(f: A => B): M[B]
```

2. **Монада Reader протаскивает конфигурацию.**

```
case class Reader[Env, A](run: Env => A):
  def map[B](f: A => B): Reader[Env, B] =
    Reader(env => f(run(env)))
  def flatMap[B](f: A => Reader[Env, B]): Reader[Env, B] =
    Reader(env => f(run(env)).run(env))
```

**Reader[Env, A]** — это обертка вокруг функции, которая ожидает на вход окружение Env. 
Монада позволяет связать множество функций в одну цепочку,
а реальный объект config мы используем всего один раз в самом конце с помощью метода .run(config).

3. **Монада Writer - чек и логирование**

```
case class Writer[A](log: List[String], value: A):
  def map[B](f: A => B): Writer[B] =
    Writer(log, f(value))
  def flatMap[B](f: A => Writer[B]): Writer[B] =
    val next = f(value)
    Writer(log ++ next.log, next.value)
```

Контейнер **Writer[A]** всегда хранит пару: список строк List[String] (накопленные логи) и полезный результат A. 
Когда две функции в банкомате соединяются через .flatMap, монада автоматически берет логи первой функции, логи второй функции и склеивает их вместе.
На выходе мы получаем и результат вычисления, и полную историю шагов для печати чека.

4. **Монада State - память банкомата**

```
case class State[S, A](run: S => (A, S)):
  def map[B](f: A => B): State[S, B] = State { s =>
    val (res, nextState) = run(s)
    (f(res), nextState)
  }
  def flatMap[B](f: A => State[S, B]): State[S, B] = State { s =>
    val (res, nextState) = run(s)
    f(res).run(nextState)
  }
```

**State[S, A]** — это обертка над функцией вида S => (A, S).
Она принимает текущее состояние банкомата (AtmState), делает его копию с новыми данными и возвращает кортеж: результат операции и новое состояние.
Метод .flatMap прокидывает это обновленное состояние на вход следующему шагу вычислений.

5. **Монада IO - управление сайд эффектами**

```
case class IO[A](unsafeRun: () => A):
  def map[B](f: A => B): IO[B] =
    IO(() => f(unsafeRun()))

  def flatMap[B](f: A => IO[B]): IO[B] =
    IO(() => f(unsafeRun()).unsafeRun())

object IO:
  def pure[A](a: A): IO[A] =
    IO(() => a)

  def delay[A](eff: => A): IO[A] =
    IO(() => eff)
```

Монада инкапсулирует внутри себя функцию () => A.
Программа не выполняет ввод-вывод сразу, а создает порядок для будущих действий.
Затем через .flatMap все действия склеиваются в единую цепочку которая выполняется в методе @main через **UnsafeRun()**
